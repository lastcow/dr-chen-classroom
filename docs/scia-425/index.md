## Course Overview

This course surveys the quality processes and technologies used in software development to assure that new software provides sufficient security for its threat environment and functions as intended. Students examine the full software assurance lifecycle — from eliciting security requirements and performing threat modeling, through architecture review, static analysis, and code inspection, to dynamic testing, fuzzing, and security-focused penetration testing. The course covers formal methods for correctness verification, statistical quality control, and metrics-driven quality management. Students also explore process assurance frameworks, DevSecOps integration, compliance testing against regulatory standards, and emerging trends that are reshaping how the industry approaches software quality and security assurance.

!!! abstract "What You Will Learn"
    By the end of this course, you will be able to:

    - Define software assurance principles and explain their role in producing secure, reliable software systems
    - Elicit, document, and evaluate security requirements and functional specifications
    - Conduct structured threat modeling exercises to identify and prioritize risks early in the development lifecycle
    - Apply quality-driven thinking to software architecture and design decisions
    - Perform static analysis and code inspection to identify defects and security vulnerabilities before execution
    - Design and execute systematic testing plans using fundamental and advanced testing techniques
    - Apply dynamic testing methods, including fuzzing, to uncover runtime vulnerabilities
    - Conduct security testing and penetration testing as part of a software assurance program
    - Use statistical quality control methods and metrics to evaluate and improve software quality
    - Apply formal methods and correctness verification techniques to high-assurance software components
    - Integrate security assurance practices into DevSecOps pipelines for continuous quality enforcement
    - Map software testing and assurance activities to compliance requirements and regulatory standards
    - Manage software quality programs using industry-recognized quality management frameworks
