## Course Overview

This course explores the fundamental principles of operating system security, examining how modern operating systems manage processes, memory, and resources — and where security guarantees can succeed or fail. Students study OS-level authentication mechanisms, access control models, and security policy enforcement before moving into more advanced topics such as sandboxing, isolation techniques, and software vulnerability exploitation. Both Windows and Linux security architectures are examined in depth through comparative case studies. The course also addresses virtualization security, kernel-level threats including rootkits, OS hardening using industry benchmarks, and emerging threats facing contemporary operating environments.

!!! abstract "What You Will Learn"
    By the end of this course, you will be able to:

    - Explain the security implications of core OS mechanisms including process management, memory management, and file systems
    - Analyze OS authentication mechanisms and evaluate their strengths and weaknesses
    - Apply access control models — including DAC, MAC, and RBAC — within operating system contexts
    - Describe and implement security policy enforcement mechanisms in both Windows and Linux environments
    - Explain sandboxing and isolation techniques and their role in containing software threats
    - Identify and analyze common software vulnerabilities at the operating system level
    - Compare the security architectures of major operating systems through structured case studies
    - Assess virtualization platforms for security risks and appropriate mitigations
    - Detect and respond to kernel-level threats including rootkits and privilege escalation
    - Apply OS hardening guidelines based on CIS Benchmarks and industry best practices
