The Biba Integrity Model

Core explanation

The Biba model , developed by Kenneth Biba in 1977, is the logical complement to Bell LaPadula, focusing on integrity rather than confidentiality.

Classroom interpretation

It uses a similar structure of subjects with integrity levels and objects with integrity levels, but with inverted rules: 1.

The Clark Wilson Integrity Model

Core explanation

The Clark Wilson model , proposed by David Clark and David Wilson in 1987, takes a different approach to integrity, grounded in commercial business practices rather than military classification systems.

Classroom interpretation

It recognizes that integrity in a business context means something specific: data should only be modified through authorized, auditable procedures that maintain internal consistency.

Discretionary Access Control (DAC)

In Discretionary Access Control (DAC) , the owner of a resource controls access to it.

DAC is flexible and intuitive, but it has a fundamental weakness: once a subject has been granted access, they may be able to pass that access to others (through copying files, changing permissions, or other mechanisms).From the Week 10 reading material

Core explanation

In Mandatory Access Control (MAC) , access control is determined by system policy rather than resource owners' discretion.

Classroom interpretation

MAC systems implement the Bell LaPadula model (or variants of it) in practice.

B

Reading Material

Attribute Based Access Control (ABAC)

Core explanation

ABAC is significantly more powerful than RBAC — it can express fine grained, context sensitive policies that RBAC cannot easily represent — but it is more complex to manage and reason about.

Classroom interpretation

ABAC is implemented through policy languages like XACML (eXtensible Access Control Markup Language) and is increasingly used in cloud platforms (AWS IAM policies are essentially ABAC) and zero trust architectures.

The Principle

Core explanation

No implicit trust : No user, device, or network location is inherently trusted, regardless of whether they are "inside" the network.

Classroom interpretation

Verify explicitly : All access must be authenticated and authorized using all available signals: user identity, device health, location, time, behavior analytics.

CIS Controls

Core explanation

The CIS Critical Security Controls (CIS Controls) , maintained by the Center for Internet Security, are a prioritized set of 18 security controls derived from analysis of the most common attack patterns.

Classroom interpretation

They are organized into three Implementation Groups (IGs) of increasing maturity: IG1 (Basic Cyber Hygiene) : Controls 1 6, addressing inventory, patching, access control, and awareness.
