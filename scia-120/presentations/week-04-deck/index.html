The System Call Interface as Attack Surface

Core explanation

Every time a user space application needs to do something privileged — read a file, open a network connection, allocate memory — it must issue a system call (syscall).

Classroom interpretation

The syscall interface is a carefully managed boundary between unprivileged code and the kernel.

4.2 The Principle of Least Privilege

Core explanation

The Principle of Least Privilege (PoLP) states that any user, process, or system component should operate with only the minimum permissions necessary to perform its legitimate function — no more, no less.

Classroom interpretation

The principle has profound implications for system design and administration: A web server process should not run as root (or SYSTEM on Windows) — it needs only to read web files and accept network connections.

Mandatory Access Control (MAC)

Core explanation

Mandatory Access Control removes discretion from resource owners.

Classroom interpretation

Instead, a central security policy (enforced by the OS or a security module) determines access based on labels assigned to both subjects (users, processes) and objects (files, resources).

4.4 Process Isolation and Memory Protection

Core explanation

Modern operating systems implement process isolation to prevent one process from interfering with another.

Classroom interpretation

Each process runs in its own virtual address space — the OS provides each process with the illusion that it has the machine's entire memory to itself, mapped through a Memory Management Unit (MMU) in hardware.
